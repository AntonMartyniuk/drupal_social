Regions

We have split up the fluid layout of the interface into regions to which in Drupal we can assign blocks to. The following regions are used. When you scroll down we see how these regions are placed in layouts.

Header

Primary navigation, user navigation, site search

Breadcrumbs

Pathway navigation for large screen and 'go back' navigation for mobile

Hero

Provides a summary of the main content of this page, for example a person, a group or an event.

Secondary navigation

Navigation options to the sub contents of a main content.

Main content

Content about this section

Complementary

Complementary content, filters, actions, quick navigations that specific to the section

Footer

It is static and contains information about the whole application.

Layout A | Parents

This layout is used on the section's parent pages. The parents form the landing pages of the platform such as the timeline page of a person or group or the detail page of an event. These pages always have child pages.

Layout B | Children

This layout is used on section's child pages. For example an event node has enrollment as a child, but a topic node doesn't have children. So we use this template for event enrollment, but not for topic.

Please note that the difference here is only seen on smaller screens compared to Layout A. The hero area will be hidden and instead of the secondary navigation there is a "Back to main page" button which leads back to Layout A.

Layout C | Orphans

This layout is used on sections that has no sub content nor has sibling pages in the hierarchy. For example a topic node, or and edit node page. Compared to the other layouts we do not have a secondary navigation nor title region here. The title will be displayed in the Hero region.
